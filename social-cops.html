Confronting the world's most critical problems through data intelligence

Problem Statement :

We're designing a massive data infrastructure that can process through millions of data points by scraping data from various sources - PDF, web and open data. A challenge when dealing with massive data sets is making this data searchable and consumable.
Google revolutionized the way we search the web. But even Google doesn't allow us to search through data sets for particular data points :)
We want to build a searchable data engine, and we are using elastic search to make data searchable by the data point.

Some of the inherent challenges here are:

  • A single data point such as "Households with an improved source of drinking water facility (%)" has different values for the same time period according to different surveys (such as NHFS, Census, and CES).

  • Users could be looking for data in different ways — by data source, by data topic, by category, and by the data point. How do you make the search experience as easy and useful for the user?

  • Data differ by geography, time period and other factors. Different surveys could be responsible for collecting data on district/ state/ national/ global levels.

Keeping all these challenges in mind, I've to think about how best we can solve this problem and design the world's best search and search results pages for data!
